# Проект: Прогнозирование кредитного риска в BNPL-транзакциях

## 📌 О проекте

Цель проекта — разработать модель машинного обучения для прогнозирования дефолта клиентов в сервисе «Покупай сейчас, плати потом» (BNPL).  
На основе финансовых и поведенческих характеристик клиентов модель оценивает вероятность невыплаты кредита.

---

## 📊 Данные

Датасет содержит 10 345 записей и 18 признаков.  
Целевая переменная — `default_flag` (1 — дефолт, 0 — оплачено).  
Доля дефолтов — около 39 % (умеренный дисбаланс).

### Основные признаки:
- `age`, `employment_type`, `monthly_income`, `credit_score`
- `purchase_amount`, `product_category`, `bnpl_installments`
- `repayment_delay_days`, `missed_payments`
- `debt_to_income_ratio`, `risk_score` (композитный, не используется в модели из-за утечки данных) и другие.

---

## 🛠 Предобработка и EDA

- Проверка пропусков и дубликатов — отсутствуют.
- Выбросы не удалялись, так как ансамблевые модели устойчивы к ним.
- Визуализация распределений и корреляционный анализ.
- Категориальные признаки закодированы методом one‑hot encoding.
- Временные признаки (`transaction_date`) преобразованы в месяц, день недели, выходной/будний, но позже исключены из‑за слабого влияния.
- По статистическим тестам (χ², Манна–Уитни) отброшены неинформативные признаки: `location`, `product_category`, `bnpl_installments`, `app_usage_frequency`, `order_month`, `order_year`, `is_weekend`, `customer_segment`.

---

## 🧠 Создание новых признаков

Для улучшения предсказательной способности добавлены следующие признаки:

- `purchase_to_income` — отношение суммы покупки к доходу.
- `install_to_income` — доля одного ежемесячного платежа в доходе.
- `debt_to_credit` — отношение долговой нагрузки к кредитному рейтингу.
- `delay_times_missed` — произведение дней просрочки на число пропусков платежей.
- `income_to_credit` — отношение дохода к кредитному рейтингу.

## 🤖 Моделирование

### Метрика качества
Основная метрика — **PR‑AUC** (Precision‑Recall AUC), так как классы несбалансированы.  
Дополнительно: F1‑score, Recall, Precision.

### Испытанные модели
- Логистическая регрессия (с подбором гиперпараметров)
- Random Forest
- XGBoost
- LightGBM
- CatBoost
- Ансамбли: **Voting (soft)** и **Stacking** (с оптимизацией порога)
- CalibratedClassifierCV (пробовала использовать калибровку вероятностей)

Все модели обучались с использованием `StratifiedKFold` (5 фолдов) и подбором гиперпараметров через `GridSearchCV`.

## 📈 Результаты

| Модель                | PR‑AUC | F1    | Recall | Precision |
|-----------------------|--------|-------|--------|-----------|
| Logistic Regression   | 0.6781 | 0.6066| 0.6374 | 0.5787    |
| Random Forest         | 0.7144 | 0.6453| 0.9233 | 0.4960    |
| XGBoost               | 0.7124 | 0.6197| 0.7129 | 0.5480    |
| LightGBM              | 0.7062 | 0.6135| 0.6993 | 0.5464    |
| CatBoost              | 0.7130 | 0.6182| 0.7005 | 0.5533    |
| **Voting (soft)**     | **0.7159** | 0.6351| 0.7636 | 0.5436    |
| Stacking              | 0.7130 | 0.6218| 0.6869 | 0.5681    |
| Stacking (opt)        | 0.7130 | 0.6563| 0.9963 | 0.4894    |

**Лучшая модель по PR‑AUC** — **Voting (soft)** (0.7159).  
Она даёт наилучшее ранжирование вероятностей дефолта.

---

## ✅ Выводы

- Созданные признаки (`delay_times_missed`, `purchase_to_income` и др.) оказались полезными и вошли в топ по корреляции с целевой переменной.
- Ансамблирование (Voting) превзошло отдельные модели по PR‑AUC.
- Random Forest показал высокий Recall (0.92), но низкую Precision — множество ложных срабатываний.
- Логистическая регрессия — самая интерпретируемая, но уступает ансамблям в качестве ранжирования.

**Рекомендуемая модель для бизнес‑задач** — **Voting (soft)**, обеспечивающая лучший компромисс между полнотой и точностью.
